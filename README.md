## Описание проекта

Проект содержит набор ручных и автоматизированных тестов для проверки пользовательского функционала формы регистрации на странице:

[https://demoqa.com/register](https://demoqa.com/register)

Автоматизированные тесты реализованы с использованием:

* Python 3.10+
* Pytest
* Selenium WebDriver
* ChromeDriver
* Архитектурный паттерн: Page Object Pattern

---

## Структура проекта

```
pages/      – Page Object'ы
tests/      – автоматизированные тесты
utils/      – вспомогательные утилиты и тестовые данные
```

### Pages

* `base_page.py` — базовый класс страницы с общими методами
* `register_page.py` — Page Object страницы регистрации

### Tests

* `test_registration_positive.py` — позитивные сценарии регистрации
* `test_registration_validation.py` — негативные и валидационные сценарии

---

## Автоматизированные сценарии

В проекте автоматизированы следующие тест-кейсы:

* TC-01 — Успешная регистрация с валидными данными
* TC-06 — Валидация логина (логин начинается с цифры)
* TC-09 — Минимальная длина пароля

Выбор сценариев обусловлен их критичностью, стабильностью и высокой ценностью при регрессионном тестировании.

---

## Особенности тестирования CAPTCHA

Механизм защиты от ботов (CAPTCHA) не автоматизируется, так как:

* CAPTCHA предназначена для предотвращения автоматических действий
* автоматизация CAPTCHA противоречит её назначению

В автотестах проверяется невозможность отправки формы без прохождения защиты.

## Установка виртуального окружения
Предполагаем, что python уже установлен

```bash
python -m venv .venv
```
---
## Активация вирутального окружения
Windows 
```bash
.venv\Scripts\activate
```

Linux
```bash
source .venv/bin/activate
```
---

## Установка зависимостей

```bash
pip install -r requirements.txt
```

---

## Запуск тестов

Запуск всех автотестов:

```bash
pytest
```

Запуск с отображением браузера:

```bash
pytest -s
```
