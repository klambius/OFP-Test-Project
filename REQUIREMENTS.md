## Ревью требований: замечания и вопросы

### Общие замечания

1. Не указано, являются ли поля **First Name** и **Last Name** обязательными.
2. Не определены допустимые длины для полей:

   * First Name
   * Last Name
   * UserName (минимальная и максимальная длина)
3. Отсутствует описание поведения формы при ошибках валидации (текст ошибок, момент появления).
4. Не описано поведение формы при успешной регистрации (редирект, сообщение, автоматический логин).

### Вопросы по логину (UserName)

1. Допускаются ли:

   * только латинские буквы в нижнем регистре?
   * логины с цифрами не в конце?
2. Что происходит при вводе логина, который уже существует:

   * ошибка на клиенте?
   * ошибка после отправки формы?
3. Есть ли ограничение на максимальную длину логина?

### Вопросы по паролю

1. Есть ли требования к сложности пароля (обязательное наличие цифры / спецсимвола)?
2. Есть ли ограничение на максимальную длину пароля?
3. Должен ли пароль отображаться как `•••••` при вводе?
4. Должно ли быть дополнительное поле подтверждения пароля? (best practice)

### Защита от ботов

1. Какой именно механизм используется:

   * CAPTCHA
   * reCAPTCHA
   * иной?
2. Можно ли отправить форму без прохождения защиты?

### Кнопка Back to login

1. Нечетко сформулировано: страница входа это:
    * главная страница?
    * страница ввода логина и пароля?
2. Теряются ли введённые данные при возврате?
